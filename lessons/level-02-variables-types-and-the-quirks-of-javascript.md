# Level 2: Variables, types, and the quirks of JavaScript

**Track:** JavaScript · **Level:** 2/12 · **Cycle:** 2 · **Difficulty:** `beginner`

📚 **Today's lesson** — published 2026-07-05

## TL;DR

JavaScript has 8 types but you'll use 5-6 daily: string, number, boolean, null, undefined, object, array, symbol, bigint. Unlike most languages, JavaScript is dynamically typed — a variable can hold any type, and you can change it.

## Real-world analogies

- A JavaScript variable is a magic box that can hold anything. The box doesn't care if you put a number in, then take it out and put a string in. The box just holds what you last put in.
- `null` vs `undefined`: `undefined` is 'I haven't decided yet' (default). `null` is 'I deliberately set this to nothing'.

## Key concepts

### ``typeof``

Tells you the type of a value: `typeof 42 === 'number'`. Famous gotcha: `typeof null === 'object'` — a 25-year-old bug that's kept for compatibility.

### `Type coercion`

JavaScript silently converts types in many places: `"5" - 3 === 2` (string becomes number). `"5" + 3 === "53"` (number becomes string). The `+` operator is special — it concatenates if either side is a string.

### `NaN`

"Not a Number". You get it from invalid math: `0/0`, `Number("hello")`. Weirdly, `typeof NaN === 'number'`. Use `Number.isNaN(x)` to check.

### `Truthy and falsy`

Only `false`, `0`, `-0`, `0n`, `""`, `null`, `undefined`, and `NaN` are falsy. EVERYTHING else is truthy — including `[]`, `{}`, and `"false"`.

```
// === Typeof — checking types ===
typeof 42;              // "number"
typeof "hello";         // "string"
typeof true;            // "boolean"
typeof undefined;       // "undefined"
typeof null;            // "object"  ← famous bug, never fixed
typeof {};              // "object"
typeof [];              // "object"  ← arrays are objects
typeof function(){};    // "function"

// === Type coercion (the dangerous magic) ===
"5" + 3;       // "53" (number → string, then concatenate)
"5" - 3;       // 2 (string → number, then subtract)
"5" * "2";     // 10 (both → numbers)
true + 1;      // 2 (true → 1)
false + 1;     // 1 (false → 0)
null + 1;      // 1 (null → 0)
undefined + 1; // NaN (undefined can't be a number)

// === Number gotchas ===
0.1 + 0.2;     // 0.30000000000000004 (floating-point math, not JS-specific)
NaN === NaN;   // false (NaN is not equal to itself!)
Number.isNaN(NaN);  // true (use this)

// === Truthy and falsy ===
Boolean("");        // false (empty string)
Boolean("hello");   // true (non-empty string)
Boolean(0);         // false
Boolean(-1);        // true (any non-zero number is truthy)
Boolean([]);        // true (empty array is truthy!)
Boolean({});        // true (empty object is truthy!)
Boolean(null);      // false
Boolean(undefined); // false
Boolean(NaN);       // false

// === == vs === (always use ===) ===
5 == "5";      // true (loose: coerces string to number)
5 === "5";     // false (strict: different types)
null == undefined;  // true (loose: special case)
null === undefined; // false (strict: different types)

// === Implicit returns in arrow functions ===
const double = (x) => x * 2;  // No return keyword, no braces
const add = (a, b) => {
    return a + b;             // Block body needs explicit return
};

// === Default parameters ===
const greet = (name = "World") => `Hello, ${name}!`;
greet();          // "Hello, World!"
greet("Aby");     // "Hello, Aby!"

// === Destructuring ===
const user = { name: "Aby", age: 30, city: "Trivandrum" };
const { name, age } = user;  // name = "Aby", age = 30
console.log(`${name} is ${age}`);  // "Aby is 30"

// Array destructuring
const colors = ["red", "green", "blue"];
const [first, second] = colors;  // first = "red", second = "green"

// Spread operator
const a = [1, 2, 3];
const b = [4, 5, 6];
const merged = [...a, ...b];  // [1, 2, 3, 4, 5, 6]

const obj1 = { x: 1 };
const obj2 = { y: 2 };
const combined = { ...obj1, ...obj2 };  // { x: 1, y: 2 }
```

## Common pitfalls

- ⚠️ Using `==` instead of `===`. Always use `===` unless you have a specific reason.
- ⚠️ Confusing `null` and `undefined`. Use `null` when you intentionally have no value, `undefined` when a value is missing.
- ⚠️ Trusting `typeof` for `null` — it's broken. Use `x === null` to check for null.
