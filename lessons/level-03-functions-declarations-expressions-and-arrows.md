# Level 3: Functions: declarations, expressions, and arrows

**Track:** JavaScript · **Level:** 3/12 · **Cycle:** 2 · **Difficulty:** `beginner`

📚 **Today's lesson** — published 2026-07-06

## TL;DR

Functions are reusable blocks of code. JavaScript has 3 ways to write them: function declarations, function expressions, and arrow functions. Each has subtle differences in how they handle `this`, hoisting, and brevity.

## Real-world analogies

- A function is a recipe. You write it once (the definition). Every time you 'cook' (call the function), you follow the same steps with different ingredients (arguments).
- Arrow functions are shorthand for the chef who's in a hurry. They skip the formalities (no `this` binding, no `arguments` object) and get straight to cooking.

## Key concepts

### `Function declaration`

`function foo() {}` — hoisted (can call before definition). Has its own `this`.

### `Function expression`

`const foo = function() {}` — NOT hoisted. Has its own `this`.

### `Arrow function`

`const foo = () => {}` — NOT hoisted. Inherits `this` from surrounding scope. Shorter syntax.

### `First-class functions`

Functions are values. Pass them as arguments, return them from other functions, store them in variables. This is what makes async code, callbacks, and React possible.

## Code with comments

Every line has a comment. Read it slowly.

```
// === Function declaration (hoisted) ===
// You can call this BEFORE its definition because it's hoisted
sayHello("Aby");  // works!

function sayHello(name) {
    return `Hello, ${name}!`;
}

// === Function expression (not hoisted) ===
// Calling before definition throws an error
// greet("Aby");  // ReferenceError!

const greet = function(name) {
    return `Hello, ${name}!`;
};

greet("Aby");  // works now

// === Arrow function (concise, inherits this) ===
const greet2 = (name) => `Hello, ${name}!`;

// Single parameter: parens optional
const greet3 = name => `Hello, ${name}!`;

// No parameters: parens required
const sayHi = () => "Hi!";

// Multiple statements: need braces and return
const processUser = (user) => {
    const greeting = `Hello, ${user.name}`;
    return greeting.toUpperCase();
};

// === Default parameters ===
const greet4 = (name = "World", greeting = "Hello") => `${greeting}, ${name}!`;
greet4();              // "Hello, World!"
greet4("Aby");         // "Hello, Aby!"
greet4("Aby", "Hi");   // "Hi, Aby!"

// === Rest parameters ===
const sum = (...numbers) => numbers.reduce((total, n) => total + n, 0);
sum(1, 2, 3, 4);  // 10

// === Functions as values (callbacks) ===
const numbers = [1, 2, 3, 4, 5];
const doubled = numbers.map(n => n * 2);     // [2, 4, 6, 8, 10]
const evens = numbers.filter(n => n % 2 === 0);  // [2, 4]
const sum2 = numbers.reduce((acc, n) => acc + n, 0);  // 15

// === `this` — the confusing part ===
// Regular function: `this` is the calling object
const obj = {
    name: "Aby",
    regular: function() { return this.name; },      // "Aby"
    arrow: () => this.name,                         // undefined (arrow inherits outer `this`)
};
obj.regular();  // "Aby"
obj.arrow();    // undefined (or whatever `this` is in the surrounding scope)

// === Higher-order functions: functions that return functions ===
const multiplier = (factor) => (number) => number * factor;
const double = multiplier(2);
const triple = multiplier(3);
double(5);  // 10
triple(5);  // 15

// === IIFE — Immediately Invoked Function Expression ===
// Runs as soon as it's defined
(function() {
    console.log("I run immediately!");
})();

// === Method shorthand in objects ===
const user = {
    name: "Aby",
    greet() {                    // Same as greet: function()
        return `Hi, I'm ${this.name}`;
    }
};
```

## Common pitfalls

- ⚠️ Using arrow functions as object methods when you need `this`. Arrow functions don't have their own `this`.
- ⚠️ Confusing function declaration (hoisted) with function expression (not hoisted). Calling the latter before definition throws.
- ⚠️ Forgetting that arrow functions inherit `arguments` from the surrounding scope — they don't have their own.
