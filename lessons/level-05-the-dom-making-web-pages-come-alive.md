# Level 5: The DOM: making web pages come alive

## TL;DR

The DOM (Document Object Model) is how JavaScript sees your HTML page — as a tree of objects you can read and modify. Click a button, change text, add a class — that's all DOM manipulation.

## Real-world analogies

- The DOM is a family tree of your HTML. `<body>` is the parent, `<h1>` and `<p>` are children. JavaScript can walk this tree, change nodes, add new ones, or remove existing ones.
- Think of the DOM as a document you can edit with JavaScript. `document.querySelector` is the 'find' function. Once you have an element, you can change its text, style, or behavior.

## Key concepts

### `Selecting elements`

`document.querySelector('#id')` and `document.querySelectorAll('.class')` are the modern way. `querySelector` returns the first match, `querySelectorAll` returns all.

### `Modifying content`

`.textContent` (plain text), `.innerHTML` (HTML — careful!), `.value` (form inputs), `.style` (CSS), `.classList` (add/remove CSS classes).

### `Events`

`.addEventListener('click', handler)` runs `handler` when the element is clicked. Other events: `submit`, `input`, `keydown`, `mouseover`, `load`.

### `Event delegation`

Instead of attaching a listener to every list item, attach ONE to the parent and check what was clicked. Much more efficient for long lists.

```
// === Selecting elements ===
const heading = document.querySelector("h1");           // First <h1>
const allButtons = document.querySelectorAll("button"); // All <button>s
const byId = document.getElementById("greeting");       // By ID
const byClass = document.getElementsByClassName("card");// By class (live list)

// === Modifying content ===
heading.textContent = "New title";          // Plain text
heading.innerHTML = "<em>New</em> title";  // HTML (use sparingly — XSS risk!)

// === Modifying styles ===
heading.style.color = "red";
heading.style.fontSize = "24px";  // camelCase, not font-size

// === Modifying classes ===
heading.classList.add("highlight");
heading.classList.remove("highlight");
heading.classList.toggle("active");
heading.classList.contains("active");  // true/false

// === Attributes ===
const link = document.querySelector("a");
link.getAttribute("href");
link.setAttribute("href", "https://example.com");
link.removeAttribute("target");

// === Creating elements ===
const newDiv = document.createElement("div");
newDiv.textContent = "Hello!";
newDiv.className = "card";
document.body.appendChild(newDiv);  // Add to end of body

// === Removing elements ===
newDiv.remove();  // Easy!
// Or: parent.removeChild(newDiv);

// === Events ===
const button = document.querySelector("#myButton");
button.addEventListener("click", (event) => {
    console.log("Button clicked!");
    console.log(event.target);  // The element that was clicked
});

// Common events: click, submit, input, change, keydown, keyup,
// mouseover, mouseout, focus, blur, load, resize, scroll

// === Form handling ===
const form = document.querySelector("#myForm");
form.addEventListener("submit", (event) => {
    event.preventDefault();  // Stop the form from reloading the page
    const formData = new FormData(form);
    const name = formData.get("name");
    const email = formData.get("email");
    console.log({ name, email });
});

// === Event delegation (efficient for long lists) ===
const list = document.querySelector("#todoList");
list.addEventListener("click", (event) => {
    if (event.target.matches(".delete-btn")) {
        event.target.closest("li").remove();
    }
});

// === Modifying data attributes ===
element.dataset.userId;          // Reads data-user-id
element.dataset.userId = "42";   // Sets data-user-id="42"

// === Animation: requestAnimationFrame ===
function animate() {
    // Update animation
    requestAnimationFrame(animate);  // Schedule next frame
}
requestAnimationFrame(animate);
```

## Common pitfalls

- ⚠️ Using `innerHTML` with user input. Always use `textContent` for untrusted data to prevent XSS attacks.
- ⚠️ Attaching event listeners inside loops — creates memory leaks. Use event delegation instead.
- ⚠️ Forgetting `event.preventDefault()` on form submit. The page reloads, losing all your state.
