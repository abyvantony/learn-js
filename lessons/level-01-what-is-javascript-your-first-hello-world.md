# Level 1: What is JavaScript? Your first 'Hello World'

**Track:** JavaScript · **Level:** 1/12 · **Difficulty:** `beginner`

📚 **Today's lesson** — published 2026-07-03

## TL;DR

JavaScript is the language of the web browser. Every interactive thing on a webpage — popups, animations, form validation, dynamic content — runs JavaScript. It also runs on servers (Node.js), in databases (MongoDB), and on IoT devices.

## Real-world analogies

- HTML is the skeleton of a webpage (structure). CSS is the skin and clothes (appearance). JavaScript is the muscles and brain (behavior and interactivity).
- JavaScript is the only programming language that runs natively in every web browser. That's why it's the most popular language in the world.

## Key concepts

### `Interpreted, not compiled`

Browsers read JavaScript line by line, no build step needed. Modern JS engines (V8, SpiderMonkey) compile it just-in-time for speed.

### `Two environments`

Browser: runs on the user's machine, can manipulate the page. Node.js: runs on the server, has access to the filesystem and network.

### ``script` tag`

JavaScript goes between `<script>` tags in HTML, or in a separate `.js` file referenced with `src`. Put it at the end of `<body>` so the page loads first.

### ``console.log``

JavaScript's `echo` — prints to the browser's developer console (F12 to open). Your best friend for debugging.

## Code with comments

Every line has a comment. Read it slowly.

```
// This is a single-line comment. JavaScript ignores it.

/*
This is a multi-line comment.
Useful for explaining tricky code.
*/

// console.log prints to the browser's developer console
// Open DevTools (F12) to see the output
console.log("Hello, world!");

// Variables: let, const, var
// Modern JS: use const by default, let if you need to reassign, never var
const name = "Aby";        // const = cannot be reassigned
let age = 30;              // let = can be reassigned
var oldStyle = "avoid me"; // var = old, function-scoped, has weird quirks

// Strings, numbers, booleans
const greeting = "Hello";           // string
const count = 42;                   // integer
const price = 19.99;                // float
const isLawyer = true;              // boolean
const nothing = null;               // null (intentionally empty)
const notDefined = undefined;       // undefined (not assigned)

// Template literals (backticks) — embed variables in strings
const message = `Hello, ${name}! You are ${age} years old.`;
console.log(message);  // "Hello, Aby! You are 30 years old."

// Basic math
console.log(2 + 3);       // 5
console.log(10 / 3);      // 3.333...
console.log(10 % 3);      // 1 (remainder)
console.log(2 ** 3);      // 8 (2 to the power of 3)

// Comparison
console.log(5 > 3);       // true
console.log(5 === "5");   // false (strict: different types)
console.log(5 == "5");    // true (loose: coerces types — usually avoid this)

// HTML example: putting JS in a webpage
// <!DOCTYPE html>
// <html>
//   <head><title>My first JS</title></head>
//   <body>
//     <h1 id="greeting">Hello</h1>
//     <script>
//       const name = prompt("What's your name?");
//       document.getElementById("greeting").textContent = `Hello, ${name}!`;
//     </script>
//   </body>
// </html>

// Run JS in Node.js (server-side, no browser needed):
// $ node hello.js
```

## Try it yourself

Create an `index.html` file with a button. When clicked, use `console.log` to print 'Button clicked!'. Open the file in a browser, open DevTools (F12), and click the button.

## Common pitfalls

- ⚠️ Using `var` instead of `let`/`const`. `var` has function-scoping and hoisting quirks that cause subtle bugs.
- ⚠️ Putting `<script>` in `<head>` without `defer` or `async` — the script runs before the HTML loads, breaking DOM access.
- ⚠️ Forgetting that `==` does type coercion. Always use `===` (strict equality) unless you really need coercion.
